# AI 兽设推算手机端 PRD

## 1. 产品概述

### 1.1 产品定位

本产品是一款偏娱乐测试的 AI 兽设生成手机端产品。用户通过快速问卷或深度输入提供性格、审美、物种、世界观和角色偏好，系统推演出专属兽设，并一次性输出完整形象图、多维度设定图和设定说明。

核心体验不是让用户直接写 prompt，而是让用户感觉自己完成了一次“兽设人格推算”。系统根据答案生成一个有物种依据、视觉特征、故事背景和任务设定的角色。

### 1.2 一句话描述

输入你的性格、审美和幻想偏好，AI 推演并生成你的专属兽设。

### 1.3 目标用户

- 轻度娱乐用户：喜欢人格测试、二次元、头像、社交分享。
- OC 用户：希望生成原创角色设定、头像、角色参考图。
- 约稿用户：需要给画师看的清晰设定说明和禁改项。
- AI 绘图用户：希望用更低门槛得到稳定 prompt 和角色设定。

### 1.4 首版目标

P1 的目标是验证三件事：

- 用户是否愿意完成问卷或深度输入。
- 用户是否认可推演出的兽设“像自己”。
- 用户是否愿意保存、分享或导出最终结果。

首版不做社区、AI 聊天、复杂角色宇宙和完整局部重绘。首版只做一个完整闭环：输入 -> 推演 -> 三产物输出 -> 保存/分享。

## 2. 产品范围

### 2.1 P1 范围

P1 必做：

- 快速算兽设：8-12 道选择题。
- 深度定制：多维度参数输入。
- 血统模式：AI 推荐、纯血、混血。
- 物种推演系统：规则引擎 + LLM。
- 结构化设定生成。
- 一次性输出三类结果：
  - 角色完整形象图。
  - 角色多维度设定图。
  - 设定说明。
- 结果页展示。
- 设定卡保存/分享。
- 基础导出：图片导出、文本复制。

### 2.2 P2 范围

P2 再做生成后的细粒度微调：

- 更可爱、更凶、更神秘、更赛博、更国风。
- 更毛茸茸、更像龙、减少人类特征、增强动物特征。
- 换配色、换尾巴、换服装。
- 保留设定只重画图。
- 多版本管理。
- 局部重绘。
- 角色进化：初始形态、成长形态、觉醒形态、暗堕形态。

### 2.3 暂缓范围

- 社区信息流。
- 角色关系网。
- AI 聊天人格。
- Live2D / VTuber 模型。
- 复杂约稿市场。
- 完整世界观编辑器。

- 禁止事项。

## 5. 血统模式设计

血统模式是 P1 核心控制项，必须放在生成前，并在结果页允许用户切换一次重新推演。

```text
血统模式
[AI 推荐] [纯血] [混血]
```

### 5.1 AI 推荐

默认模式。系统根据问卷判断用户更适合纯血还是混血。

- 偏经典、明确、辨识度高：推荐纯血。
- 偏幻想、复杂、独特、OC 感强：推荐混血。
- 选择赛博、神话、异化、强幻想风格时，提高混血权重。

### 5.2 纯血模式

纯血模式只输出一个主物种。

输出规则：

- 物种纯度为 100%。
- 不输出混合比例。
- 强化主物种辨识度。
- 禁止加入其他物种的显著特征。
- 设定图中标记为「纯血」。

示例：

```text
血统：纯血
物种：雪豹
重点：长尾、斑纹、厚毛、山地猎手气质
禁止：不要加入龙角、狐尾、羽翼、机械异化
```

### 5.3 混血模式

混血模式输出 2-3 个物种或幻想标签，并明确比例和对应身体特征。

输出规则：

- 必须有一个主血统。
- 主血统比例建议不低于 55%。
- 副血统用于局部特征或世界观异化。
- 不允许副血统压过主血统。
- 每个血统必须落到具体视觉部位。

示例：

```text
血统：混血
主血统：赤狐 65%
副血统：东方龙 25%
异化：机械义体 10%
狐系特征：狐耳、大尾巴、灵活体型
龙系特征：肩颈龙鳞、竖瞳、尾端火焰纹
机械特征：耳侧发光纹路、义体护臂
```

## 7. AI 能力链路

### 7.1 总链路

```text
用户输入
↓
答案标签化 / 关键词提取
↓
兽设打分引擎
↓
LLM 生成 character_spec_json
↓
Prompt 生成器
↓
并行生成三类结果
  ├─ 完整形象图
  ├─ 多维度设定图
  └─ 设定说明
↓
结果页统一展示
↓
保存 / 分享 / 导出
```

### 7.2 模块分工

| 能力 | 模块 |
| --- | --- |
| 问卷理解 | LLM + 标签规则 |
| 物种推荐 | 规则引擎 + LLM |
| 血统判断 | 规则引擎 |
| 设定文案 | LLM |
| Prompt 生成 | LLM |
| 图片生成 | 文生图模型 |
| 一致性控制 | 统一 character_spec_json、seed、参考图 |
| 设定图排版 | 前端模板或图像生成 |
| 设定说明 | 结构化渲染 |
| JSON 存档 | 后端结构化保存 |

### 7.3 同源生成原则

完整形象图、多维度设定图和设定说明必须从同一份 `character_spec_json` 派生。禁止三个结果各自重新理解用户输入，避免物种、配色、体型和故事互相矛盾。

## 8. 输出设计

### 8.1 一次生成三类结果

P1 必须一次性输出：

1. 角色完整形象图。
2. 角色多维度设定图。
3. 设定说明。

用户不需要分三次点击生成。

### 8.2 完整形象图

用途：情绪价值、分享、结果页主视觉。

内容：

- 角色全身或半身主体。
- 背景故事对应的世界场景。
- 当前任务设定。
- 明确的角色状态和氛围。

示例：

```text
赤狐纯血夜行斥候，站在霓虹雨夜的废城屋顶，正在执行追踪任务。
```

### 8.3 多维度设定图

用途：OC 保存、约稿参考、角色设定沉淀。

内容：

- 正面角色图。
- 背面或侧面简化参考。
- 局部特征：耳朵、尾巴、眼睛、爪子、角、鳞片、羽毛等。
- 表情或动作参考。
- 配色盘。
- 标志性道具。
- 血统标识：纯血或混血比例。
- 禁止事项。

### 8.4 设定说明

用途：解释推演逻辑、给用户保存、给画师参考。

字段：

- 角色名。
- 血统模式。
- 物种 / 混合比例。
- 身高 / 体型。
- 性格关键词。
- 视觉关键词。
- 主色 / 辅色 / 点缀色。
- 耳朵、尾巴、爪子、毛发或鳞片特征。
- 服饰风格。
- 标志性道具。
- 背景故事。
- 当前任务设定。
- 角色口头禅。
- 必须保留项。
- 禁止事项。
- 头像 prompt。
- 参考图 prompt。

## 10. 页面设计

### 10.1 首页

核心文案：

```text
输入你的性格、审美和偏好，AI 帮你推演专属兽设。
```

入口：

- 30 秒算兽设。
- 深度定制。
- 已有兽设优化，P2。

### 10.2 问卷页

移动端一屏一题。

结构：

- 顶部进度。
- 中部问题。
- 底部选项按钮。
- 可跳过非关键题。
- 最后一题后进入血统模式选择。

### 10.3 深度定制页

结构：

- 血统模式。
- 物种偏好。
- 世界观风格。
- 体型和特征强度。
- 配色。
- 必须保留项。
- 禁止事项。

### 10.4 生成中页面

生成过程需要有推演感。

展示步骤：

- 正在解析人格倾向。
- 正在匹配物种血统。
- 正在生成世界身份。
- 正在绘制完整形象。
- 正在整理设定图和说明。

### 10.5 结果页

移动端建议用纵向完整页面：

```text
角色名 / 血统 / 物种
↓
完整形象图
↓
多维度设定图
↓
设定说明
↓
保存 / 分享 / 导出
```

结果页需要支持：

- 查看完整图片。
- 查看设定图。
- 复制设定说明。
- 保存图片。
- 分享设定卡。
- 切换纯血/混血重新推演一次。

## 11. 视觉方向

### 11.1 设计关键词

- 神秘幻想。
- 精致卡片。
- 手机测试感。
- OC 设定板。
- 轻赛博与轻魔法结合。

### 11.2 推荐风格

整体风格采用深色沉浸式界面，辅以冷青、雾紫、暖金或赤橙点缀。界面应像“人格测试 + 角色档案 + 幻想设定卡”的结合。

避免：

- 太幼稚的儿童插画感。
- 纯工具表单感。
- 过重的游戏后台感。
- 低质 AI 图堆叠感。

### 11.3 移动端样式结构

- 首页：沉浸式主视觉 + 两个明确入口。
- 问卷：一屏一题，按钮大，进度明确。
- 血统选择：三段式控件，纯血/混血解释清楚。
- 结果页：图片优先，文字说明可折叠。
- 设定图：像角色参考表，而不是普通海报。

## 12. 关键指标

P1 重点看：

- 问卷开始率。
- 问卷完成率。
- 生成成功率。
- 结果保存率。
- 分享点击率。
- 纯血/混血切换率。
- 深度定制入口点击率。
- 用户是否复制设定说明。

## 13. 风险与约束

### 13.1 图片一致性风险

风险：完整形象图和多维度设定图看起来不是同一个角色。

解决：

- 使用同一份 `character_spec_json`。
- 统一主物种、配色、体型和必须保留项。
- 生成设定图时强调 reference sheet。
- P2 再引入 seed、参考图和局部重绘。

### 13.2 物种混乱风险

风险：混血后主物种不清晰。

解决：

- 主血统比例不低于 55%。
- 每个副血统必须落到具体局部。
- 增加禁止项：不要让副物种压过主物种。

### 13.3 文案空泛风险

风险：设定说明像模板，不像用户。

解决：

- 问卷必须收集关系、行为、防御机制和审美偏好。
- 设定说明要解释“为什么推演成这个兽设”。
- 避免只输出形容词堆叠。

## 14. 首版验收标准

P1 可验收条件：

- 用户可以完成快速问卷并生成结果。
- 用户可以完成深度定制并生成结果。
- 用户可以选择 AI 推荐、纯血或混血。
- 系统能一次性输出完整形象图、多维度设定图和设定说明。
- 结果页中三类产物来自同一份设定，物种、配色、体型不冲突。
- 用户可以保存或分享结果。
- 用户可以复制设定说明。
