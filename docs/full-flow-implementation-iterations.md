# AI 兽设推算全流程代码落地迭代关系

本文档用于梳理当前代码和目标全流程之间的差距，并定义后续代码落地顺序。产品不再按 demo / MVP / P2 拆分，而是按“全流程可用”拆成连续迭代。

## 1. 全流程可用定义

用户应能完成以下完整链路：

```text
首页选择入口
  ↓
快速生成：固定 Q12 隐晦问卷
或
深度定制：从本地题库按分支抽题
  ↓
选择 AI 推荐 / 纯血 / 混血
  ↓
本地规则引擎完成标签分析、物种候选、血统判断、冲突检测
  ↓
OpenAI 生成统一 character_spec_json
  ↓
并行生成完整形象图、多维度设定图
  ↓
结果页展示图片、设定说明、比例、保留项、禁止项、prompt
  ↓
用户可保存图片、复制设定、重新推演
```

### 3.4 冲突检测

当前没有落实。

需要补：

## 4. 迭代关系

### I1. 题库代码化

目标：把题库从文档变成代码数据源。

改动：

- 新增 `quickQuestions.ts`
- 新增 `deepQuestionBank.ts`
- 新增基础类型 `Question`, `QuestionOption`, `QuestionBranch`
- 快速问卷页面改为读取固定 Q12
- 深度题库先只接入基础分支，不急着完成全部交互

完成标准：

- 快速入口显示 12 题
- 题目和选项不再写死在 `page.tsx`
- 测试能验证快速题库数量固定为 12

### I2. 评分引擎统一

目标：所有模式都走同一套本地评分。

改动：

- 新增 `scoreAnswers()`
- 替换 `optionScores`
- 输出标签分、物种候选、血统建议、命中分支
- `inferCharacterBlueprint()` 改用评分快照

完成标准：

- 单个答案不会直接锁定物种
- 纯血结果没有副物种比例
- 混血结果有比例和局部映射
- 测试覆盖典型狐龙、纯血雪豹、机械混血场景

### I3. 深度分支问卷

目标：深度定制从表单变成题库分支流程。

改动：

- 新增深度问卷步骤状态
- 基础题答完后计算下一组分支题
- 单次答题数量受模式控制
- 保留少量手填项作为最后补充，不作为主要输入

完成标准：

- 深度定制能从本地题库抽题
- 不同答案会进入不同分支
- 单次题量不会超过配置上限
- 最终请求带 `answers` 和 `scoreSnapshot`

### I4. 生成前确认与冲突检测

目标：降低生成成本浪费，避免明显错误结果。

改动：

- 新增设定预览页
- 新增 `detectConflicts()`
- 纯血 / 混血 / 禁止项 / 世界观冲突自动修正或提示
- 用户确认后再调用 `/api/generate`

完成标准：

- 生成前可看到系统推断摘要
- 冲突可读且有默认修正方案
- 用户确认后进入生成中

### I5. 图片重试

目标：让结果页更像真实可用工具。

改动：

- 新增 `/api/regenerate-image`
- 单图失败可重试

完成标准：

- 任意一张图失败不影响设定说明
- 用户能单独重画完整形象图或设定图

### I6. 文档与启动说明同步

目标：让 README 与真实功能一致。

改动：

- README 去掉“原型”口径
- 更新本地题库、Q12、深度分支说明
- 更新真实生成和无 key 预览差异
- 补充常见问题

完成标准：

- 新用户按 README 能跑通本地
- 不会误以为 fallback 示例就是完整生成

## 5. 建议执行顺序

```text
I1 题库代码化
  ↓
I2 评分引擎统一
  ↓
I3 深度分支问卷
  ↓
I4 生成前确认与冲突检测
  ↓
I5 图片重试
  ↓
I6 README 同步
```

这几个迭代不是版本分级，而是依赖关系。I1 和 I2 是后续所有功能的基础；I3 让深度定制真正可用；I4 到 I5 解决真实生成成本和失败恢复问题。结果持久化暂不进入本轮执行计划。
